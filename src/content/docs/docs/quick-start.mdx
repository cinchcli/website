import { Steps, Aside, Tabs, TabItem } from '@astrojs/starlight/components';

This guide walks you through installing Cinch, authenticating, and moving your first clip end-to-end.

<Steps>

1. **Install the CLI**

   <Tabs>
   <TabItem label="macOS">
   [Download the Desktop app](https://cinchcli.com/#install) (Apple Silicon, recommended), or install the CLI via Homebrew:
   ```bash
   brew install cinchcli/tap/cinch
   ```
   </TabItem>
   <TabItem label="Linux">
   ```bash
   curl -fsSL https://cinchcli.com/install.sh | sh
   ```
   This installs the `cinch` binary to `/usr/local/bin`. Supports Debian/Ubuntu and RHEL/Fedora.
   </TabItem>
   <TabItem label="Windows">
   Download `cinch_Windows_x86_64.zip` from [GitHub Releases](https://github.com/cinchcli/cinch/releases/latest), extract, and add `cinch.exe` to PATH.
   </TabItem>
   <TabItem label="Docker (relay only)">
   ```bash
   docker run -d -p 8080:8080 -v cinch-data:/data \
     ghcr.io/cinchcli/relay:latest
   ```
   </TabItem>
   </Tabs>

   Verify the install:
   ```bash
   cinch --version
   ```

2. **Sign in**

   ```bash
   cinch auth login
   ```

   A browser window opens. Sign in with GitHub or Google.

   <Aside type="note">
   To use a self-hosted relay instead of the hosted service, pass `--relay`:
   ```bash
   cinch auth login --relay https://your-relay.example.com
   ```
   </Aside>

3. **Push your first clip**

   On machine A:

   ```bash
   echo "hello from $(hostname)" | cinch push
   # → ✓ Pushed 28 B
   ```

4. **Pull it on the other end**

   On machine B (also signed in):

   ```bash
   cinch pull
   # → hello from my-server
   ```

   Or pipe directly to the clipboard:

   ```bash
   cinch pull | pbcopy   # macOS
   cinch pull | xclip    # Linux (X11)
   ```

5. **Pair a remote machine (optional)**

   To set up cinch on a remote server in one command — SSH in, install, and pair:

   ```bash
   cinch pair user@remotehost
   ```

</Steps>

## What's next?

- Install the [desktop app](https://github.com/cinchcli/desktop) to have every push land in your Mac clipboard automatically.
- See the full [CLI Reference](/docs/cli/push/) for all flags.
- [Self-host the relay](/docs/relay/self-hosting/) if you want your data on your own server.
